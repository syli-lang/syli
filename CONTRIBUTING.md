# Contributing

Welcome and thank you for your interest in contributing to this project.

## Ways to Contribute

- Opening an issue to report a bug
- Adding new functionality
- Fixing typos and improving documentation
- Adding optimizations
- Improving the type system error messages
- Adding new tests to validate a pattern

## Before Opening a PR

Open an issue to discuss your plan before starting a pull request, unless the change is minor (fixing a typo, improving a comment, adding tests).

## Guidelines

- Keep patches minimal and local. Avoid broad refactors when fixing bugs.
- Match the existing code style (naming, formatting, conventions).
- Add comments only for complex logic; keep them short.

## AI-Generated Code

Avoid pushing AI-written code without human supervision. AI tends to introduce boilerplate and unnecessary code — review and trim any AI-generated contributions carefully.
